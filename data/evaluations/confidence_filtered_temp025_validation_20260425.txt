Key finding:
  Moderate sharpening with policy_temperature=0.25 does not improve validation top-1 accuracy over one-hot on confidence-filtered data.
  It does keep validation policy loss slightly lower at late epochs, but the difference is not enough to solve generalization.
  Confidence filtering remains the main source of improvement: validation top-1 increases from about 4-5% to about 10-13%.

Interpretation:
  The current dataset contains too few clear/high-confidence samples, and both one-hot and moderately sharpened targets still overfit.
  The next bottleneck is data quality/quantity, not simply target temperature.

Conclusion:
  min_policy_confidence=0.3 is useful but insufficient alone.
  policy_temperature=0.25 is not clearly better than one-hot for validation top-1 on the filtered subset.

Recommended next steps:
  1. Generate more high-confidence teacher data, either more games or more simulations per move.
  2. Add sample weighting by policy confidence/entropy rather than hard filtering.
  3. Add game-level validation split to check whether random-sample split is hiding game correlation effects.
  4. If training another candidate, select checkpoints by validation or arena evaluation, not train loss.
