## 阶段 7a：BoxesZero 反向训练 ✅ 已完成

参考论文: BoxesZero: An Efficient and Computationally Frugal Dots-and-Boxes Agent (Entropy 2025)

### 核心算法

**反向训练 (Backward Training)**: 从终局开始学习，逐步向开局推进
- **阶段1 (Value Network Reinforcement)**: st=50→0，每步 MCTS 自对弈只搜 st 之后的步骤
  - st 之前: 贪心启发式 (优先吃格>安全边>危险边)
  - st 之后: MCTS 搜索 + 收集数据
  - 价值目标: root Q (MCTS Q值)
  - 晋级条件: mqr >= threshold 或 迭代次数 >= mit
- **阶段2 (Policy Network Reinforcement)**: st=0，全程 MCTS
  - 价值目标: 0.25×Q + 0.75×z (混合 Q 值和最终胜负)

### 实现文件

| 文件 | 说明 |
|---|---|
| `src/backward_selfplay_main.cpp` | 反向自对弈 CLI |
| `src/AI/az/az_selfplay.h/cpp` | 新增 `runBackward()`, `playOneGameBackward()`, `rootQ` |
| `train/backward_training.py` | 编排脚本: 自对弈→训练→导出→晋级 |
| `train/dataset.py` | 新增 `value_mode` (margin/wdl/q/q+z) |
| `train/train.py` | 新增 `--value_mode`, `--q_weight` 参数 |

### 用法

```bash

./build_onnx/backward_selfplay 100 400 25 data/backward model.onnx

# 自动编排 (从 st=50 开始, 每轮30局, 每5步晋级)
cd train
python backward_training.py \
  --arch resnet_s \
  --games_per_iter 50 \
  --simulations 400 \
  --epochs_per_iter 30 \
  --start_st 50 --beta 5 --mit 3
```

## 阶段 7b：Bug 修复 + Arena 评估 + GUI 部署 ✅ 已完成（2025-05-18）

### 发现并修复的 Bug

1. **AlphaZeroMove 回合逻辑错误** (`az_move.cpp`)
   - 原问题: MCTS 落子后即使吃到格子，外层也强制换手
   - 修复: `AlphaZeroMove` 内部循环，吃到格子时同一玩家继续决策
   - 影响范围: GUI AI + evaluate_ai

2. **evaluate_ai 完全确定性** (`evaluate_ai_main.cpp`)
   - 原问题: 无 Dirichlet 噪声 + temperature=0 → 每局结果完全相同
   - 修复: 完全重写 `playOneGame`，支持 `--sims`/`--temp`/`--verbose`
   - 前几步加入轻微噪声 (alpha=0.3, frac=0.15) + 温度采样 (T=0.5)

3. **backward_training.py 数据覆盖** 
   - 原问题: 同一 st 的多轮 jsonl 文件名相同，复制到 combined 时被覆盖
   - 修复: 使用 `iter_{N:04d}__{filename}` 唯一命名
   - 注: 原 iter15 模型实际只用了 11779 样本（意外形成 sliding-window，效果反而不差）

4. **SFML ABI 不兼容** 
   - 原问题: 项目自带 SFML DLL (2022/GCC 12) 与当前 GCC 15.2 的 libstdc++ 不兼容
   - 修复: 从 GitHub 拉取 SFML 2.6.2 源码，用 GCC 15.2 重新编译
   - CMakeLists.txt 指向 `3rdparty/SFML_new`

- 退化集中在执黑能力，说明模型学到了不良先后手偏置

### 根因

1. `backward_training.py` 每轮无条件覆盖 `backward_model.onnx`，没有 arena 晋级门槛。
2. `train.py` 的 best 选择把非评估 epoch 的 train loss 与 val loss 混比，可能保存过拟合模型。
3. sample split 下验证集也随机增强，验证指标不够稳定。
4. 纯最近 8 轮 st=0 sliding window 丢掉了早期 curriculum 锚点数据。
5. Stage2 `q+z` 的 WDL 标签在白方优势明显的数据上会强化颜色/先后手偏置。

### 已实施修复

- official 模型已恢复到 `backward_st0_iter15_best_model.pt` 导出的 ONNX。
- 退化模型已备份为 `backward_model_degraded_iter45_*.onnx` 和 `backward_best_model_degraded_iter45_*.pt`。
- 新训练默认使用：
  - `--stage2_value_mode q+margin`
  - `--split_mode file`
  - `--anchor_size 15`
  - 默认开启 arena gate：candidate 总胜率、执黑胜率、执白胜率都达标才 promote
  - `--target_total_iterations` 控制总目标轮数，避免恢复后超跑。
  - 已修复回退到 iter15 后误吸入磁盘上 iter16-45 退化数据的问题：训练窗口只收集 `<= current total_iterations` 的目录，重跑当前 iter 前会清空旧目录。
